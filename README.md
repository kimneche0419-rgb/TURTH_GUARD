# TruthGuard SDK: AI 콘텐츠 신뢰성 평가 오픈소스 프레임워크

> **TruthGuard SDK**는 생성형 AI 시대에 급증하는 허위정보, 가짜뉴스, 딥페이크, AI 생성 콘텐츠를 탐지하고 그 신뢰성을 정량적으로 평가할 수 있는 개발자 지향형 오픈소스 프레임워크입니다. 본 문서는 시스템 아키텍처, 상세 기능 사양, 코드 레벨의 API 활용 예시를 포함한 프로젝트 명세서입니다.

---

## 1. 프로젝트 개요

### 1.1 프로젝트 소개
생성형 AI(Generative AI) 기술의 대중화로 대량의 콘텐츠가 정교하게 자동 생성되는 시대가 도래했습니다. 이에 따라 악의적인 가짜뉴스 양산, 딥페이크를 활용한 금융·사회적 사기, 정교한 허위 사실 유포 등의 부작용이 심화되고 있습니다.

**TruthGuard SDK**는 일반 사용자를 위한 단일 서비스가 아닙니다. 뉴스 플랫폼, SNS, 교육 시스템, AI 챗봇 등 다양한 도메인의 개발자가 자사 인프라와 서비스에 **콘텐츠 검증 레이어**를 손쉽게 임베딩할 수 있도록 지원하는 **오픈소스 SDK 및 CLI 도구**입니다.

### 1.2 핵심 아키텍처 원칙
* **Modular & Pluggable:** 텍스트, 이미지, 영상, 오디오 모듈이 완전히 분리되어 필요한 모듈만 선택적으로 로드하여 사용할 수 있습니다.
* **Model Agnostic:** 내부 탐지 백엔드는 경량 자체 모델부터 외부 거대 모델(Hugging Face, 외부 API 등)까지 유연하게 교체할 수 있습니다.
* **Explainable AI (XAI):** 단순 결과 수치(Pass/Fail) 제공을 넘어, 개발자와 사용자가 납득할 수 있는 판단 근거와 신뢰 지표를 구조화된 데이터(JSON)로 반환합니다.

## 2. 개발 배경 및 목적

### 2.1 사회적 및 기술적 문제점
1. **AI 생성 허위정보 확산:** LLM을 이용해 그럴듯하지만 사실이 아닌 정보(Hallucination)를 대량으로 자동 생성하여 유포하는 사례 폭증.
2. **미디어 변조 및 딥페이크 악용:** 초실사 이미지·영상 합성 기술을 활용한 사기 및 조작 증가.
3. **통합 검증 프레임워크의 부재:** 현재 공개된 오픈소스는 특정 기능(단순 가짜뉴스 분류, 단일 딥페이크 탐지 등)에 파편화되어 있어, 상용 서비스 플랫폼에 통합 적용하기 어렵고 접근성이 낮음.

### 2.2 개발 목적
TruthGuard SDK는 고도화된 미디어 분석 파이프라인을 표준화된 통합 API로 묶어 제공함으로써, 누구나 쉽게 **'AI 콘텐츠 안전망'**을 최소한의 비용으로 신속하게 구축할 수 있도록 돕는 것을 목적으로 합니다.

---

## 3. 핵심 목표
* **종합 미디어 탐지 SDK 제공:** 텍스트, 이미지, 비디오, 오디오를 아우르는 다각도 신뢰성 검증.
* **다차원 정량 스코어링:** 콘텐츠별 신뢰도 점수 및 위험도 수치화.
* **설명 가능성(Explainability) 확보:** 탐지 결과의 원인이 되는 근거(탐지된 특징점, 문맥적 오류 지점 등)를 명확히 제공.
* **개발자 친화적 API & 플러그인 구조:** 쉽게 도입할 수 있는 API와 플러그인 확장 구조 지원.

## 4. Quick Start (빠른 시작)

TruthGuard SDK 및 대시보드를 신속하게 실행하는 방법입니다.

### 4.1 설치 및 환경 설정 (Installation & Setup)

프로젝트 저장소를 로컬에 복제하고 필요한 백엔드(Python) 및 프론트엔드(Node.js) 의존성을 구성합니다:

**Step 1. 저장소 복제 및 디렉터리 이동**
```bash
git clone https://github.com/kimneche0419-rgb/TURTH_GUARD.git
cd TURTH_GUARD
```

**Step 2. 파이썬 가상환경 생성**
```bash
python -m venv .venv
```

**Step 3. 백엔드 의존성 및 패키지 개발자 모드 설치 (기본/AI 텍스트 분석 포함)**
```bash
# Windows
.venv\Scripts\pip install -e .[text]

# macOS / Linux
.venv/bin/pip install -e .[text]
```

**Step 4. 프론트엔드 대시보드 라이브러리 설치**
```bash
npm install
```


### 4.2 기본 환경 초기화
작업 공간 내에 설정 파일(`truthguard.json`)과 미디어 수집 폴더(`uploads/`)를 준비합니다:
* **PowerShell:** `.\tg init`
* **CMD:** `tg init`

### 4.3 개별 및 통합 실행 명령어
가상환경 활성화 없이 루트 경로에서 즉각 구동할 수 있는 단축 명령어를 제공합니다:

| 명령어 (CMD) | 명령어 (PowerShell) | 설명 | 주요 특징 및 옵션 |
|:---|:---|:---|:---|
| `tg init` | `.\tg init` | **프로젝트 환경 초기화** | `truthguard.json` 및 `uploads/` 폴더 생성. 기존 설정 초기화 시 `--force` 적용 |
| `tg dev` | `.\tg dev` | **통합 개발 서버 실행** | 백엔드 API(8000) 및 프론트엔드 대시보드(5173)를 각각 다른 새 창으로 동시 구동 (추천) |
| `tg api` | `.\tg api` | **백엔드 API 서버 단독 실행** | FastAPI 서버를 현재 세션에서 실행 (`--port <포트>`, `--host <호스트>` 옵션 지원) |
| `tg web` | `.\tg web` | **대시보드 웹 서버 단독 실행** | 프론트엔드 대시보드를 현재 세션에서 단독 실행 |
| `tg cli <파일/URL>` | `.\tg cli <파일/URL>` | **CLI 미디어/웹 신뢰도 분석 (단축)** | `tg scan` 명령어의 단축 별칭으로 터미널에서 신속하게 분석 |
| `tg scan <파일/URL>` | `.\tg scan <파일/URL>` | **CLI 미디어/웹 신뢰도 분석** | 파일 또는 웹사이트 주소를 분석하여 보고서를 출력합니다. (`-f json`, `-f table` 및 `--threshold` 지원) |
| `tg mcp` | `.\tg mcp` | **MCP Stdio 표준 서버 실행** | LLM Agent(예: Claude)와 연동하기 위해 stdio 기반 JSON-RPC로 대화하는 MCP 서버 구동 |

## 6. 프로젝트 디렉토리 구조
```plaintext
truthguard/
 ├── text/           # 텍스트 허위정보 및 AI 생성 탐지
 ├── image/          # 이미지 변조 및 딥페이크 탐지
 ├── video/          # 비디오 프레임 및 템포럴 분석
 ├── audio/          # 오디오 합성 탐지 (확장 예정)
 ├── explain/        # 판단 근거 추출 및 XAI 포맷팅 엔진
 ├── cli/            # 명령줄 인터페이스 로직
 ├── models/         # 로컬 구동용 경량 모델 파일 및 메타데이터
 ├── datasets/       # 테스트 및 평가용 데이터셋
 ├── docs/           # 상세 개발자 설계 문서 및 가이드
 ├── tests/          # 단위/통합 테스트 코드
 └── examples/       # 활용 예제 (REST API 등)
```
